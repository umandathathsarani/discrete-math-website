Question 1: The Core Definition

What is the fundamental rule that makes a relation a valid Function?

🪄 Lumos! Reveal Answer

Answer:
Every element in the Domain (input) must map to exactly one unique element in the Codomain (output). An input cannot shoot out two arrows!

Question 2: Domain vs. Range

In a function f: X → Y, what is the difference between the Codomain and the Range?

🪄 Lumos! Reveal Answer

Answer:
The Codomain is the entire set Y (all *possible* outputs). The Range is the specific subset of Y containing only the elements that actually *get mapped to* by elements in X.

Question 3: Calculating Range

Let the Domain be X = {-2, -1, 0, 1, 2}. If the function is f(x) = x², what is the Range?

🪄 Lumos! Reveal Answer

Answer:
Range = {0, 1, 4}.
(-2² = 4, -1² = 1, 0² = 0, 1² = 1, 2² = 4). Notice how duplicates are only listed once in a set!

Question 4: Equality of Functions

Let f(x) = x + 2 and g(x) = (x² - 4) / (x - 2). Both map Real numbers to Real numbers. Are f and g equal functions?

🪄 Lumos! Reveal Answer

Answer:
No. Even though (x² - 4) / (x - 2) simplifies to x + 2, the function g(x) is undefined at x = 2 (because dividing by zero is impossible). Since their valid domains are not perfectly identical, the functions are not equal.

Question 5: Identifying Injective

Is the function f(x) = 3x + 1 (from Real numbers to Real numbers) One-to-One (Injective)? Why?

🪄 Lumos! Reveal Answer

Answer:
Yes. Every unique x value produces a completely unique y value. If f(a) = f(b), then 3a + 1 = 3b + 1, which means a = b.

Question 6: Injective Failure

Give a mathematical example of a function that is NOT One-to-One (Injective) on the set of all Real numbers.

🪄 Lumos! Reveal Answer

Answer:
f(x) = x² or f(x) = |x| (Absolute value).
In both cases, two different inputs (like 2 and -2) map to the exact same output (4 or 2), breaking the One-to-One rule.

Question 7: Identifying Surjective

Let f(x) = x² from the set of Real numbers to the set of Real numbers. Is this function Onto (Surjective)?

🪄 Lumos! Reveal Answer

Answer:
No. For a function to be Onto, the Range must equal the Codomain. The Codomain is all Real numbers, but the Range of x² is only positive numbers and zero. Negative numbers are never hit.

Question 8: Fixing Surjectivity

How could you change the Codomain of f(x) = x² to make it a Surjective (Onto) function?

🪄 Lumos! Reveal Answer

Answer:
Change the Codomain to be the set of all non-negative Real numbers ([0, ∞)). If the Codomain perfectly matches the Range, the function becomes Surjective!

Question 9: The Perfect Match

What two conditions must a function meet to be called Bijective?

🪄 Lumos! Reveal Answer

Answer:
It must be BOTH One-to-One (Injective) AND Onto (Surjective).

Question 10: Inverse Pre-requisites

True or False: Any mathematical function can be reversed to create an Inverse Function (f⁻¹).

🪄 Lumos! Reveal Answer

Answer:
False. A function can ONLY have an inverse if it is Bijective (One-to-One and Onto). If it is not Bijective, reversing it will break the fundamental rules of a function.

Question 11: Calculating the Inverse I

Find the inverse function of f(x) = 5x - 7.

🪄 Lumos! Reveal Answer

Answer:
1. y = 5x - 7
2. y + 7 = 5x
3. x = (y + 7) / 5
4. Swap x and y: f⁻¹(x) = (x + 7) / 5

Question 12: Calculating the Inverse II

Find the inverse function of g(x) = (x / 3) + 2.

🪄 Lumos! Reveal Answer

Answer:
1. y = (x / 3) + 2
2. y - 2 = x / 3
3. 3(y - 2) = x
4. Swap x and y: g⁻¹(x) = 3(x - 2) or 3x - 6

Question 13: Many-to-One

If a function maps 3 different domain elements to the exact same codomain element, what type of function is it?

🪄 Lumos! Reveal Answer

Answer:
A Many-to-One function. (And because it is Many-to-One, it is automatically NOT One-to-One).

Question 14: Restricting the Domain

The function f(x) = x² does not have an inverse because it is Many-to-One. How can we "fix" the Domain so that the inverse f⁻¹(x) = √x is mathematically legal?

🪄 Lumos! Reveal Answer

Answer:
We must restrict the Domain to only allow positive numbers and zero (x ≥ 0). By removing the negative numbers, we remove the duplicates, making it One-to-One!

Question 15: Function Combinations

Is it possible for a function to be Onto (Surjective) but NOT One-to-One (Injective)? If yes, describe how.

🪄 Lumos! Reveal Answer

Answer:
Yes. This happens if every single item in the output gets hit by an arrow (Onto), but at least one output gets hit by *two* arrows (Many-to-One). This often occurs if the Domain is larger than the Codomain.

Question 16: Cardinality and Injective

If there is an Injective (One-to-One) function from set A to set B, what must be true about their sizes (cardinalities)?

🪄 Lumos! Reveal Answer

Answer:
Set A must be smaller than or equal to Set B. (|A| ≤ |B|).
(If A had more elements than B, they couldn't all have a unique partner without doubling up!)

Question 17: Cardinality and Surjective

If there is a Surjective (Onto) function from set A to set B, what must be true about their sizes?

🪄 Lumos! Reveal Answer

Answer:
Set A must be larger than or equal to Set B. (|A| ≥ |B|).
(You need at least enough arrows to hit every target in B).

Question 18: Cardinality and Bijective

Based on the previous two questions, if a function is Bijective, what is the relationship between the size of Domain A and Codomain B?

🪄 Lumos! Reveal Answer

Answer:
They must be exactly equal! (|A| = |B|).

Question 19: Inverse Property

If you take an input x, pass it through the function f, and then pass that result through the inverse function f⁻¹, what is your final answer? (i.e., What is f⁻¹(f(x))?)

🪄 Lumos! Reveal Answer

Answer:
The answer is just x. The inverse function exactly undoes whatever the original function did, bringing you right back to your starting point.

Question 20: Evaluating Mappings

A mapping has 3 inputs (A, B, C) and 3 outputs (1, 2, 3). The arrows are: A → 2, B → 2, C → 3.
Is this a valid function? If yes, classify it (Injective, Surjective, etc.).

🪄 Lumos! Reveal Answer

Answer:
Yes, it is a valid function (every input shoots one arrow).
It is Many-to-One (A and B both point to 2).
It is NOT Surjective (Output '1' never gets hit).
Therefore, it is also NOT Bijective.
