Lecture 7: Graphs and Algorithms

1. Introduction to Graphs

A Graph is a fundamental data structure in computer science consisting of a set of vertices (nodes) and a set of edges (connections between the nodes).

Graphs are a pervasive data structure, and algorithms for working with them are fundamental to the field. Hundreds of interesting computational problems are defined in terms of graphs.

Figure 1: A simple graph representation.

Real-World Applications

  • Road Networks: Cities act as vertices, and roads between cities act as edges.
  • Electronic Circuits: Junctions act as vertices, and electronic components (resistors, capacitors) act as edges.
  • Aviation/Flights: Cities act as vertices, and flight paths from one city to another act as edges.

2. Graph Categorization

  • Directed Graphs (Digraph): All the edges have a specific direction. They point from one vertex to another.
  • Undirected Graphs: Edges are not directed. The connection is a two-way street.
  • Weighted Graphs: All the edges have been assigned a numerical weight (e.g., cost, distance, time).
  • Multigraphs: The same pair of vertices have more than one edge connecting them.

Figure 2: Examples of basic graph classifications.

3. Graph Terminology

A graph is formally defined as G = (V, E) where V is the set of vertices and E is the set of edges. Edges in a directed graph are ordered pairs.

  • Orientation: In a directed graph, the edge (i, j) is different from (j, i). The orientation is strictly from i to j.
  • Adjacent Vertices: If (i, j) is an edge of the graph, vertex i and vertex j are adjacent. In a digraph, vertex j is adjacent from i, and i is adjacent to j.
  • Incident: An edge is incident to the vertices it connects. In a digraph (i, j), vertex j is incident from i.
  • Loop or Self-edges: An edge (i, i) from a vertex to itself.
  • Degree (d(v)): The number of edges incident to a vertex v. In directed graphs:
    • Indegree: The number of incoming edges at the vertex.
    • Outdegree: The number of outgoing edges from the vertex.

Figure 3: Node degrees in a directed graph.

4. Paths, Circuits, and Cycles

  • Path: A sequence of vertices <v0, v1, v2, ..., vk> such that there is an edge between consecutive vertices.
    Example: 2, 1, 4, 5 is a path from 2 to 5.
  • Length of a Path: The number of edges in the path. In a weighted graph, it's the sum of the weights of the edges on the path.
  • Simple Path: A path where all vertices in the path are completely distinct (no vertex is visited twice).
  • Reachable: Vertex A is said to be reachable from B if there is a path from B to A.
  • Circuit: A path whose first and last vertices are the exact same.
  • Cycle: A simple circuit. Except for the first and last vertex (which are the same), no other vertex appears more than once.
  • Hamiltonian Cycle: A cycle of a graph G that contains absolutely all the vertices of G.

5. Subgraphs and Trees

  • Subgraph: A subgraph of a graph G = (V, E) is a graph H = (U, F) such that U is a subset of V, and F is a subset of E.
  • Connected Graph: A graph is connected if there is at least one path from every vertex to every other vertex in the graph.
  • Tree: A connected undirected graph that contains NO cycles. It has exactly |V| - 1 edges.
  • Forest: An undirected graph that does not contain a cycle (essentially a collection of disconnected trees).
  • Spanning Tree: A subgraph of G that is a tree and contains all the vertices of G.

Figure 4: A Tree is fully connected; a Forest is a collection of disconnected trees.

6. Representation of Graphs

There are two standard ways to represent a graph G=(V,E) in computer memory. Both are applicable to directed and undirected graphs.

  • 1. Adjacency List An array of lists. Each individual list shows what vertices a given vertex is adjacent to.

    Properties:
    • For directed graphs, the sum of lengths of all lists is |E|.
    • For undirected graphs, the sum is 2|E| (since edges go both ways).
    • Memory Required: Θ(V + E). Compact and highly preferred for sparse graphs.
    • Disadvantage: Slower to determine if a specific edge (u, v) exists, as you must search the list Adj[u].
  • 2. Adjacency Matrix An n × n matrix A = (aij) where elements are 1 if an edge exists from i to j, and 0 otherwise.

    Properties:
    • Memory Required: O(V2).
    • Preferred when the graph is dense (|E| is close to |V|2) or when quick edge lookups between two vertices are needed.

7. Searching Graphs: Breadth First Search (BFS)

Why do we search? To find paths and to determine connectivity.

Given a graph G=(V,E) and a source vertex s, BFS systematically explores the edges of G to "discover" every vertex that is reachable from s.

  • It computes the shortest distance (smallest number of edges) from s to each reachable vertex.
  • It produces a "breadth-first tree" with root s.
  • Data Structure: Implemented using a Queue (First-In, First-Out).

The BFS Coloring System:

  • WHITE: Vertex is undiscovered.
  • GRAY: Vertex is discovered, but its adjacency list is currently being processed.
  • BLACK: Vertex is completely finished.

Figure 5: BFS explores the graph level-by-level, utilizing a Queue.

BFS(G, s) Algorithm Steps:
1. For each vertex u ∈ V[G] - {s}:
    color[u] = WHITE, d[u] = ∞, π[u] = NIL
2. color[s] = GRAY, d[s] = 0, π[s] = NIL
3. Q = ∅, ENQUEUE(Q, s)
4. While Q ≠ ∅:
    u = DEQUEUE(Q)
    For each v ∈ Adj[u]:
       If color[v] == WHITE:
          color[v] = GRAY
          d[v] = d[u] + 1
          π[v] = u
          ENQUEUE(Q, v)
    color[u] = BLACK

Running Time: O(V + E)

8. Searching Graphs: Depth First Search (DFS)

DFS searches the graph deeper whenever possible. Edges are explored out of the most recently discovered vertex v that still has unexplored edges. When all of v's edges have been explored, the search "backtracks".

  • The predecessor subgraph may be composed of several trees (a depth-first forest).
  • Data Structure: Implemented using a Stack (or recursion).
  • Timestamps: Each vertex v gets two timestamps:
    d[v]: First discovered (grayed).
    f[v]: Finished examining adjacency list (blackened).
DFS(G) Algorithm Steps:
1. For each vertex u ∈ V[G]: color[u] = WHITE, π[u] = NIL
2. time = 0
3. For each vertex u ∈ V[G]:
    If color[u] == WHITE, then DFS-VISIT(u)

DFS-VISIT(u)
1. color[u] = GRAY
2. time = time + 1; d[u] = time
3. For each v ∈ Adj[u]:
    If color[v] == WHITE:
       π[v] = u
       DFS-VISIT(v)
4. color[u] = BLACK
5. time = time + 1; f[u] = time

Running Time: Θ(V + E)
