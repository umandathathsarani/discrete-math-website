Lecture 2: Functions (Complete Notes)

1. Definition of a Function

A Function (f) from a set X to a set Y is a relationship between elements of X and elements of Y with the property that each element of X is related to a unique element of Y.

  • Domain: The set of all valid input values (Set X).
  • Codomain: The set from which the function's output values are chosen (Set Y).
🪄 In Simple Terms: Think of a function as a magical sorting machine. You drop an item from the Domain into the machine, and it guarantees to spit out exactly one specific item from the Codomain. It never spits out two things for one input!

2. Range / Image

The unique element y to which f sends x is denoted by f(x). This is called "f of x", the value of f at x, or the Image of x under f.

The set of all actual values of f taken together is called the Range of f (or the image of X under f).

🪄 Codomain vs. Range: The Codomain is every possible output that *could* exist. The Range is the set of outputs that actually *do* get hit by the function. The Range is always a subset of the Codomain.

3. Equality of Functions

Two functions, f: A → B and g: A → B, are considered strictly equal if and only if they satisfy two key conditions:

  1. They have the exact same Domain (A) and Codomain (B).
  2. They produce the same output for every single input. Mathematically: f = g if and only if ∀x ∈ A, f(x) = g(x).

4. Types of Functions

Functions are classified by how inputs map to outputs:

  • One-to-One (Injective) For all elements x1 and x2 in X, if f(x1) = f(x2), then x1 = x2.
    🪄 In Simple Terms: No two different inputs map to the same output. Every output gets hit by a maximum of one arrow.
  • Onto (Surjective) Given any element y in Y, it is possible to find an element x in X such that y = f(x).
    🪄 In Simple Terms: The Range equals the Codomain! Every single item in the output set gets hit by at least one arrow. Nothing is left out.
  • Many-to-One Two or more elements of the domain set are related to a single element of the codomain set.
    🪄 In Simple Terms: Multiple inputs point to the exact same output. (This is the opposite of One-to-One).
  • One-to-One Correspondence (Bijective) A function that is BOTH One-to-One (Injective) and Onto (Surjective).
    🪄 In Simple Terms: A perfect pairing. Every input has a unique output, and every output has exactly one matching input. No overlaps and no leftovers!

5. Inverse Functions (f -1)

An inverse function exists only when the function f(x) is Bijective (both one-to-one and onto).

If f: X → Y, then the inverse is f -1: Y → X.
Definition: f -1(y) = { x ∈ X | f(x) = y }

🪄 Example 1: Linear Equation

Find the inverse of f(x) = 2x + 3 (mapping real numbers to real numbers).

  1. Start with the equation: y = 2x + 3
  2. Solve for x in terms of y: x = (y - 3) / 2
  3. Interchange x with y: y = (x - 3) / 2
  4. The inverse function is: f -1(x) = (x - 3) / 2

🪄 Example 2: Quadratic Equation

Find the inverse of f(x) = x2

  1. Start with the equation: y = x2
  2. Solve for x in terms of y: x = √y
  3. Interchange x with y: y = √x
  4. The inverse function is: f -1(x) = √x
🪄 A Wizard's Warning: The function f(x) = x² is technically a "Many-to-One" function (e.g., both 2² and -2² equal 4), which means it shouldn't have an inverse! To find the inverse f⁻¹(x) = √x, we must secretly restrict the Domain to only allow positive numbers (x ≥ 0).

6. Functions in Summary (Combinations)

It is important to remember that functions can mix and match these properties. Here are the five possible scenarios when evaluating a mapping from set A to set B:

  • 1. One-to-One, but NOT Onto
    🪄 Scenario: Every input has a unique output, but there are leftover items in the Codomain that never get hit.
  • 2. Onto, but NOT One-to-One
    🪄 Scenario: Every output gets hit by an arrow, but at least one output gets hit by multiple arrows (Many-to-One).
  • 3. Both One-to-One AND Onto (Bijective)
    🪄 Scenario: The perfect 1:1 match. This is the ONLY scenario where an Inverse Function can exist!
  • 4. Neither One-to-One NOR Onto
    🪄 Scenario: Some outputs are hit by multiple arrows, AND there are still leftover outputs that never get hit at all.
  • 5. NOT a Function
    🪄 Scenario: A single input shoots out two or more arrows to different outputs. This breaks the fundamental definition of a function!
