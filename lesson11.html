Lecture 11: Finite Automata & String Matching

1. String Matching Basics

String Matching is the computational problem of finding one or more occurrences of a specific "pattern" string within another larger string or body of "text".

Example of a Shift:
Text T:    A B C A B A A B C A B A C
Pattern P:        A B A A      (This is Shift s=3)

Real-World Applications

String matching is a highly important subject in the wider domain of text processing. Its applications include:

  • Text Processing & Editing: E.g., the "Find and Replace" feature in word processors.
  • Computer Security: Scanning for specific signatures in virus detection and analyzing/checking passwords.
  • Bioinformatics: DNA sequence analysis to find specific genetic markers.
  • Data Communications: Analyzing network packet headers.

2. String Matching with Finite Automata

Many string-matching algorithms build a Finite Automaton that scans the text string T for all occurrences of the pattern P.

Efficiency: These string-matching automata are highly efficient. They examine each text character exactly once, resulting in a constant time calculation per text character, making the search incredibly fast and linear.

3. Formal Definition of a Finite Automaton

A finite automaton M is mathematically defined as a 5-tuple: (Q, q0, A, Σ, δ)

  • Q: A finite set of internal states.
  • q0: The start state, where q0 ∈ Q.
  • A: A distinguished subset of Q representing the accepting states (A ⊆ Q).
  • Σ (Sigma): A finite input alphabet (e.g., {a, b}).
  • δ (Delta): The transition function from Q × Σ into Q. It maps the current state and current input character to the next state.

4. Automata Representation & Case Study 1

Automata are typically represented either visually via a state-transition diagram or systematically via a tabular transition function.

Case Study: Accepting strings ending in an odd number of 'a's

  • State Set (Q): {0, 1}
  • Start State (q0): 0
  • Accepting State (A): 1 (often blackened or double-circled in diagrams)
  • Input Alphabet (Σ): {a, b}

Figure 1: State-transition diagram for an automaton accepting an odd number of 'a's.

Tracking the state transitions:

  • Input: abaaa → Sequence of states: (0 → 1 → 0 → 1 → 0 → 1). Since it ends in state 1 (Accepting), the automaton accepts this input.
  • Input: abbaa → Sequence of states: (0 → 1 → 0 → 0 → 1 → 0). Since it ends in state 0, it rejects this input.

5. Case Study 2: Matching the pattern "ababaca"

This is a more complex automaton designed specifically to match the string P = ababaca.

  • The Spine: Right-going directed edges forming the "spine" of the automaton correspond to successful matches between the pattern and input characters.
  • Failing Matches: Left-going edges correspond to failing matches (resetting back to an earlier state).
  • Convention: If a state i has no explicitly shown outgoing edge for some character a ∈ Σ, it is assumed that δ(i, a) = 0.

Figure 2: Automaton matching "ababaca". The final state (7) is blackened indicating acceptance.

Tracking the State Function φ(Ti)

When running this automaton on the text T = abababacaba, we track the state φ(Ti) the automaton is in after processing the prefix up to character i.

Text T:      a  b  a  b  a  b  a  c  a  b  a
State φ(Ti): 0  1  2  3  4  5  4  5  6  7  2  3

Notice it hits state 7 at index 9, meaning one occurrence of the pattern was successfully found!
