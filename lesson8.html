Lecture 8: Trees, Heaps & Minimum Spanning Trees

1. Introduction to Trees

An undirected graph is defined as a Tree if and only if there is a unique simple path between any two of its vertices. By definition, a tree contains no cycles.

2. Rooted Trees

A Rooted Tree is a tree in which one specific vertex has been designated as the root, and every edge is directed away from the root.

  • Components are named after natural trees: root, branches, and leaves.
  • Traditionally drawn with the root at the top and branching downwards.
  • Parent/Child Relationship: A node pointing down is a parent, and the nodes it points to are its children.
  • Leaf Node: A node that has no children.

Figure 1: A Rooted Tree showing Root, Branches, and Leaves.

3. Binary and Complete Binary Trees

Binary Tree: A rooted tree where every internal vertex has at most two children. The children are specifically distinguished as a left child and a right child.

Complete Binary Tree: A strict version of a binary tree where every internal vertex has exactly two children, and absolutely all leaves are at the exact same level (depth).

Figure 2: Standard Binary Tree vs. Complete Binary Tree.

4. Essential Tree Terminology

  • Degree of a Node: The number of subtrees of a node.
  • Degree of a Tree: The maximum degree of any node in the tree.
  • Ancestors & Descendants: Ancestors of a node are all nodes along the path from the root to that node. Descendants are all nodes in the subtrees originating from that node.
  • Level (Depth): The level of a node is the number of edges along the path from the root to that node. The root is at level 0.
  • Height: The maximum level of any node in the tree.

5. Application: Arithmetic Expressions

Trees are widely used in Computer Science. One major application is representing complex arithmetic expressions.

  • Internal Vertices represent mathematical operators (+, -, *, /).
  • Leaves (External Vertices) represent the operands (numbers or variables).

Figure 3: An Arithmetic Expression represented as a Tree.

6. Tree Traversals

By traversing an arithmetic expression tree in different orders, we can extract three different standard notations:

  • Pre-order Traversal (Prefix Notation): Root → Left → Right.
  • In-order Traversal (Infix Notation): Left → Root → Right.
  • Post-order Traversal (Postfix Notation): Left → Right → Root.

7. Heaps

A Heap is a specialized complete binary tree data structure that satisfies the heap property:

  • Max-Heap: Every parent node contains a value greater than or equal to the values of its children. The maximum element is always at the root.
  • Min-Heap: Every parent node contains a value less than or equal to the values of its children. The minimum element is always at the root.

8. Spanning Trees

A Spanning Tree of a simple graph G is a subgraph that is a tree and contains absolutely all the vertices of G.

9. Minimum Spanning Trees (MST) & Prim's Algorithm

In a connected, weighted graph, a Minimum Spanning Tree is a spanning tree whose sum of edge weights is as small as possible.

Prim's Algorithm

Prim's is a Greedy Algorithm used to find the Minimum Spanning Tree. It builds the tree one vertex at a time by always making the locally optimal choice.

Prim's Algorithm Steps:
1. Let V be the set of vertices in the MST, initially containing one arbitrary starting node.
2. Let E' be the set of edges in the MST, initially empty.
3. Repeat until all vertices have been chosen:
    Choose a vertex v that is NOT currently in V.
    Ensure that the edge connecting v to a vertex already inside V has the minimal possible weight (Greedy choice!).
    Add v to V, and add the edge to E'.
4. The final MST cost is the sum of all weights in E'.
