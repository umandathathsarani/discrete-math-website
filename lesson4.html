Lecture 4: The Basics of Counting (Complete Notes)

1. Introduction to Counting

Counting is the mathematical process of determining the number of elements in a set or the number of ways to arrange or choose objects without actually having to list every single one out.

Common Applications:

  • Counting outcomes in a game of chance.
  • Determining possible combinations of clothes.
  • Calculating the number of unique passwords or network IP addresses that can be created.

2. The Four Rules of Counting

To count efficiently, we rely on four fundamental frameworks depending on how the tasks are structured.

  • 1. The Addition Rule (Sum Rule) If a task can be done in one of n1 ways OR in one of n2 ways, and none of the n1 ways overlap with the n2 ways, then there are n1 + n2 total ways to do the task.
    🪄 In Simple Terms: Use the Sum Rule when the choices are Mutually Exclusive (you can only pick ONE option total from multiple different groups). Keywords: "OR".
  • 2. The Multiplication Rule (Product Rule) If a procedure can be broken down into a sequence of two tasks, where the first task can be done in n1 ways and the second task can be done in n2 ways, then there are n1 × n2 ways to do the procedure.
    🪄 In Simple Terms: Use the Product Rule when events happen Sequentially (you must do task 1 AND task 2). Keywords: "AND", "FOLLOWED BY".
  • 3. The Subtraction Rule (Inclusion-Exclusion) If a task can be done in either n1 ways or n2 ways, but some of these ways overlap (they belong to both), we must subtract the overlap so we don't count it twice.
    |A ∪ B| = |A| + |B| - |A ∩ B|
    🪄 In Simple Terms: Use this when choices overlap. Add the groups together, then subtract the common items!
  • 4. The Division Rule If a task can be done using a procedure that yields n total ways, but every distinct outcome is actually repeated d times (grouped), then the number of unique ways is n / d.

3. Permutations (Order Matters!)

A permutation is an arrangement of items in a specific, strict order. The order of the objects absolutely matters (e.g., the password "123" is completely different from "321").

Permutations are categorized into three distinct types based on the rules of the arrangement:

  • Type A: Permutation Without Repetition Each item can be used only once. The number of permutations of r elements chosen from a total of n distinct elements is given by:

    P(n, r) = n! ⁄ (n - r)!
  • Type B: Permutation With Repetition Items can be used multiple times (e.g., a lock where you can use the number '5' three times in a row). The formula is simply:

    nr
    (where n is the number of options, and r is the number of slots to fill).
  • Type C: Permutation With Indistinguishable Objects Some items in the group are perfectly identical and cannot be distinguished from each other (e.g., arranging the letters in the word "SUCCESS").

    n! ⁄ (n1! × n2! × ... × nk!)
    🪄 Formula Guide: n! is the total number of items. You divide by the factorials of the counts of each identical item.
