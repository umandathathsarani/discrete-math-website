Lecture 9: Recursion

1. Recursive Thinking

To solve a problem recursively means to find a way to break it down into smaller subproblems, each having the exact same form as the original problem.

A recursively defined picture: First, an original picture is given. Then a process of successively superimposing centered smaller pictures on top of the previous pictures is carried out.

Figure 1: Visual representation of a recursively defined picture.

2. Recursively Defined Functions

A recursive definition of a mathematical function consists of two essential steps:

  • BASIS STEP: Specify the value of the function at the least possible input integer(s).
  • RECURSIVE STEP: Give a rule for finding its value at an integer from its values at smaller integers.

Note: A function f(n) is functionally the same as a sequence a₀, a₁, ..., where f(i) = aᵢ.

3. Function Example: Simple Recurrence

Problem: Suppose f is defined by:

  • f(0) = 3
  • f(n+1) = 2f(n) + 3

Find f(1), f(2), f(3), f(4).

Solution:

  • f(1) = 2f(0) + 3 = 2(3) + 3 = 9
  • f(2) = 2f(1) + 3 = 2(9) + 3 = 21
  • f(3) = 2f(2) + 3 = 2(21) + 3 = 45
  • f(4) = 2f(3) + 3 = 2(45) + 3 = 93

4. Function Example: The Factorial Function

Problem: Give a recursive definition of the factorial function F(n) = n!.

Solution:

  • Basis Step: F(0) = 1
  • Recursive Step: F(n+1) = (n+1)F(n)

Verification for F(5):
F(5) = 5 × F(4) = 5 × 4 × F(3) = 5 × 4 × 3 × F(2) = 5 × 4 × 3 × 2 × F(1) = 5 × 4 × 3 × 2 × 1 × F(0) = 120 × 1 = 120.

5. Fibonacci Numbers

The Fibonacci sequence provides a classic example of a recursive definition where the recursive step depends on two previous values rather than just one.

Definition:

  • Basis Step: f₀ = 0 and f₁ = 1
  • Recursive Step: fₙ = fₙ₋₁ + fₙ₋₂ for n = 2, 3, 4, ...

Example: Find f₂, f₃, f₄, f₅, f₆.

  • f₂ = f₁ + f₀ = 1 + 0 = 1
  • f₃ = f₂ + f₁ = 1 + 1 = 2
  • f₄ = f₃ + f₂ = 2 + 1 = 3
  • f₅ = f₄ + f₃ = 3 + 2 = 5
  • f₆ = f₅ + f₄ = 5 + 3 = 8

6. Recursive Algorithms

Definition: An algorithm is called recursive if it solves a problem by reducing it to an instance of the exact same problem but with a smaller input.

7. Algorithm 1: Recursive Computing (Power)

Example: Give a recursive algorithm for computing aⁿ, where a is a nonzero real number and n is a non-negative integer.

Solution: Use the recursive definition of aⁿ.

procedure power(a: nonzero real number, n: nonnegative integer)
    if n = 0 then return 1
    else return a * power(a, n - 1)
{output is aⁿ}

8. Algorithm 2: Recursive GCD Algorithm

Example: Give a recursive algorithm for computing the greatest common divisor (GCD) of two nonnegative integers a and b with a < b.

Solution: Use the reduction property gcd(a, b) = gcd(b mod a, a) and the base condition gcd(0, b) = b when b > 0.

procedure gcd(a, b: nonnegative integers with a < b)
    if a = 0 then return b
    else return gcd(b mod a, a)
{output is gcd(a, b)}

9. Algorithm 3: Recursive Binary Search Algorithm

Example: Construct a recursive version of a binary search algorithm.

Solution: Assume we have an increasing sequence of integers a₁, a₂, ..., aₙ. Initially i is 1 and j is n. We are searching for element x.

Figure 3: Binary search recursively splits the array in half.

procedure binary_search(i, j, x: integers, 1 ≤ i ≤ j ≤ n)
    m := ⌊(i + j) / 2⌋
    if x = aₘ then
        return m
    else if (x < aₘ and i < m) then
        return binary_search(i, m - 1, x)
    else if (x > aₘ and j > m) then
        return binary_search(m + 1, j, x)
    else
        return 0
{output is location of x in sequence, or 0 if not found}
